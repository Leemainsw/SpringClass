# **🐧 Spring 2강**

##  어노테이션 설명
### @RestController
- RestController이 없으면 MycontrollerClass는 기능을 못 한다.
- RestController을 붙여줘야 기능을 해야하는구나라고 스프링이 알게 된다.
- 안쓴다고 에러가 나지는 않지만 동작은 안한다.

**중요한 것은 메소드**
```
@RequestMapping("/hello")
      public String hello(){
	return "<h1>Hello</h1>";
     }
```

### @RequestMapping
- 특정 주소에 대한 응답을 해줄 메소드를 정의
- ("hello")에 주소값을 주는 것임
- 이 메소드를 "핸들러"라고도 부름 (handling) -> 처리
**http://locahost:8080/hello**
 http://locahost:8080 -> RequestMapping 같은 것들이 관여 못함

localhost(127.0.0.1)
IP주소 : 컴퓨터를 구분 (0.0.0.0 ~ 255.255.255.255)
포트번호 : 프로세스를 구분 (0 ~ 65535)
Well-known ports (0 ~ 1023)
포트번호 80 : HTTP (평문)
포트번호 443 : HTTPS (암호화)

### 스프링 종료방법
1. 창 옆 빨간 네모를 눌러서 종료한다.
2. 작업관리자에서 **javaw** (JWM 실행중인 프로세스)를 종료시킨다.

Q) RequestMapping 어노테이션을 사용해서 "/bye" 주소에 접근해서 Bye라는 문자열 출력 메서드 추가하기 [서버를 띄워보고 해당 주소에 접근했을 때 잘 나오는지 확인]
A)
```
@RequestMapping("Bye")
public String bye() {
	return "<h1>Bye</h1>";
}
```

#### 어떤 주소가 있고 그 주소에 메소드를 달아두면 메소드가 요청에 대한 처리를(헨들링) 해줘서 필요한 응답을 중 수 있다. 

**위의 코드에서 인자를 추가하려면 키 값을 써야 한다.
```
@RequestMapping(value="Bye2", method=RequestMethod.GET)
	public String bye2() {
		return "<h1>Bye2</h1>";
	}
```
**일반적으로 사용하는 메소드 4가지**
get, post, put, delete

### GetMapping, PostMapping PostMan 프로그램 사용

```
@GetMapping("/hello2")
	public String hello2() {
		return "hello";//hello.html 뷰 (템플릿)을 사용
	}
```

```
@PostMapping("/post2")
        public String post2() {
	    return "<h1>post2</h1>";
}
```

#### 템플릿 엔진
