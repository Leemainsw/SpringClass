### 핸들러  메소드 반환 타입 및 Model and ModelAndView 클래스
템플릿  내부에 내용을 채워야 하는 경우에는 Model 혹은 ModelAndView 클래스의 도움을 받아야 함

**model 사용하기**
```java
@GetMapping("model")
public String model(Model m) {
	m.addAttribute("title", "Hello World 1234");
	m.addAttribute("content", "Content 1234");
	return "with_model";
}
```
-> 파라미터에서 model 값을 받는다. 
**스프링에서는 핸들러 메소드에 파라미터를 정의하면 알아서 호출하는 쪽에서 파라미터에 필요한 값을 넣어줌**

모델의 역할 => 동적 웹 페이지에 사용할 재료(데이터 보관)

#### MAP의 특징
* 키를 통해서 값을 저장 (python -> dict)
* 보통 키는 string으로 값은 object이다. 
* 값은 어떤 값이 올지 모르기 때문에 object 형식이다.

**모델은 그냥 모델 값만 채워놓으면 알아서 템플릿 엔진 쪽으로 전달함**
```java
//@RestController
@Controller //이걸로 해야 템플릿 적용이 됨
public class MyController {
```

**model and view 사용하기**
ModelAndView => 모델 + 뷰 이름 정보


```java
@GetMapping("modell_and_view")
public ModelAndView modelAndView() { // 반환 타입이 modelandview
	ModelAndView m = new ModelAndView("with_model"); // 뷰(템플릿) 이동
       //생성자에 적어주어야 하는 것은 view의 이름
	m.addObject("title", "Hello World 1234");
	m.addObject("content", "Content 1234");
	return m;
}
```
* 문자열, 단순 숫자말고 객체를 돌려줄 확률이 높음
ex) 
* 학생 정보를 볼 수 있는 페이지 -> Student 타입 객체를 모델에 저장
* 학생 정보를 볼 수 있는 페이지 -> Student 타입의 이름, 정보, 나이, 주소, 학번 등등 전부 addAttribute해서 저장

**임의의 객체를 넘기고 싶은데 공짜로 할 수 없을 때**
=> 클래스를 자바 빈(Bean) 클래스로 만들어야 함

**빈이 될 수 있는 조건**
1. 모든 필드의 접근 제어자는 private (해당 필드에 접근할 수 있는 게터, 세터 메소드를 제공)
2. 외부에서 접근 가능한(public) 기본 생성자(인자 없는 생성자(no-arg constructor))를 제공
3. Serializable 인터페이스를 구현

Q) 생성자(생성자 메소드)를 접근 제어자를 private로 정할 수 있는가?
A) private의 의미 => 외부에서 호출 불가 => 생성자 호출 불가 => 싱글텀 패턴에서 주로 활용

```java
class MyClass{
      private int a;
      GetA(){return a;}
      SetA(int a){this.a = a;}
}
```

Q) 아래의 클래스에 기본생성자가 있나?
```java
class MyClass{
      private int a;
      GetA(){return a;}
      SetA(int a){this.a = a;}

      public MyClass(int a){
             //
      }
}
````
A) 생성자가 1개라도 있으면 기본생성자는 만들어 지지 않는다. (생성자가 아무것도 없을 때만 기본 생성자 만듬)

#### Serializable 인터페이스
=> [자바직렬화1](https://woowabros.github.io/experience/2017/10/17/java-serialize.html)
=> [자바직렬화2](https://hyeonstorage.tistory.com/252)
=> 객체를 파일이나 네트워크에 저장(IO) 하기 위해서 사용하는 인터페이스
```java
class Student implements Serializable{

}
```
 Serializable 인터페이스 = > Maker 인터페이스 (내용이 없는 인터페이스)
목적 -> 타입 확인용으로 사용 (캐스팅 가능 혹은 instanceof 같은거 사용 가능)
약속용 (Serializable 인터페이스를 구현한 타입만 혹은 네트워크로 저장하기로 합시다.)
ex) Cloneable은 clone 메소드를 구현하고 있다는 것을 알려주려고 인터페이스 구현
