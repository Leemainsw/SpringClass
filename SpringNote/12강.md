## 내용협상 (Content negotiation)

- Accept 붙어있는 헤더들은 전부 내가 어떤 종류의 데이터를 받고 싶은지 쓰는 용도로 사용됨.
- 서버에서는 선호도에 따라서 다은 종류의 데이터를 줄 수도 있음
- 내가 받고 싶어하는 데이터를 협상하는 과전에 Accept 붙어있는 헤더가 사용됨
- => 실제로는 잘 이루어지지 못하는 이야기 (보안성?)
- => 지원해야 하는 MIME 타입은 엄청 여러개일 수 있는데 내가 구현을 다 못해줌
- => 요즘은 그냥 json 응답해주는 것이 끝
- => API 썻을 때 가장 응답으로 많이 돌려주는 MIME 타입 => JSON / XML
- => 성실하게 내용 협상해주려는 서버의 경우 JSON, XML 둘 다 지원하는 경우도 있음

Accept : application/json=1.0, application/xml;q=0.9

Github에서 이제는 json만 지원함

json을 더 선호하는 이유
- Less Verbose : XML은 태그 + 항상 시작태그, 닫는 태그를 써야 하고 용량이 비교적 많이 나가는 편
- Faster : XML이 파싱하는데 상대적으로 오래 걸림
- Readable :  Json이 가독성이 더 놏다
- Structure Matches the Data : 자료형 정보가 데이터 자체에 이미 포함됨
